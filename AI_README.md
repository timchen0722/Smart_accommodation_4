# 專案 AI 協同開發規範與進度

## 🤖 協同開發模式公告

本專案由多個 AI 工具（Antigravity、Claude Cowork、Claude Code、Codex）輪班開發。
各工具請務必遵守以下規範：

1. 讀取或修改程式碼前，先閱讀本檔案，並執行 `git status --short`、`git log -1 --oneline`，確認本地檔案、目前分支與最新提交狀態。
2. 工作區可能同時存在使用者或其他 AI 尚未提交的變更；不得擅自還原、刪除或覆蓋，也不要使用 `git add .` 一次加入所有檔案。
3. 撰寫程式碼請遵守下方【技術棧與架構規範】，優先沿用既有模組、元件與設計語言。
4. 修改模型、資料欄位或風險指標前，必須先追查實際資料來源、分母、校準方式與目前模型產物，不可只修改畫面文案。
5. 完成工作後，請執行與變更相稱的驗證，並更新本檔案的【當前開發進度】及【備忘錄 / 重要架構決定】。
6. 交接時應記錄修改檔案、驗證結果、未完成事項，以及是否已提交 Git；提交時只加入本次工作相關檔案。

## 🛠️ 技術棧與架構規範

- 前端／應用框架：Python 3.11+、Streamlit 多頁式應用；入口為 `index.py`，主要頁面位於 `pages/`。
- 資料處理與視覺化：pandas、NumPy、Plotly、Matplotlib、PyArrow。
- 模型與解釋：LightGBM、XGBoost、scikit-learn、SHAP、LIME、Joblib；既有模型檔與套件版本有相依性，請勿任意升級或替換。
- 狀態與資料來源：以 Streamlit session/cache、CSV／Parquet 與本地模型產物為主；目前不是以資料庫作為主要應用狀態來源。
- 程式風格：Python 使用 `snake_case`、清楚的函式命名與適量型別標註；共用邏輯應放入 `modules/`，頁面檔以流程編排與呈現為主。
- UI 文案：以繁體中文為主，沿用既有日系簡約、低彩度設計；共用色彩、字級、間距與卡片樣式集中維護於 `modules/ui_components.py`。
- 共用元件：房源詳情應優先重用 `modules/listing_detail.py`；資料載入、模型推論、象限判定及地址推估不可在頁面內各自複製一套。
- 地址限制：畫面中的「推估地址」由鄰近地標推算至道路層級，不是房源真實門牌，文案不得宣稱為精確地址。
- 模型限制：風險分級、空房率、近 90 天實際未訂房率及行政區中位數差異是不同指標；調整前必須確認欄位定義與目前演算法選擇。
- 套件限制：不可隨意引進新第三方套件；確有需要時先說明用途、影響與替代方案，再取得同意。
- 機密限制：不得提交 `.streamlit/secrets.toml`、API 金鑰、帳密或其他敏感設定。
- 驗證原則：至少執行相關 Python 語法檢查、測試與 `git diff --check`；UI 變更另以實際 Streamlit 畫面確認桌面版與窄螢幕排版。
- HTML 排版限制：用 `st.markdown(..., unsafe_allow_html=True)` 內嵌 HTML 時，字串中**不得出現空白行**（含只有空白的行），否則 Markdown 會結束 HTML 區塊、把後面縮排 4 個空白的內容當成程式碼原樣印在畫面上；共用的 HTML 產生函式（如 `listing_detail.summary_html()`）一律回傳無縮排、無空行的字串。
- 熱重載限制：修改 `modules/*.py` 後，執行中的 Streamlit **不會**重新載入該模組（症狀是 `AttributeError: ... has no attribute` 或新 CSS 沒生效），必須重啟 server，按 Rerun 無效；驗收 UI 前先確認 server 是改檔之後才啟動的。

## 🗂️ 主要檔案與責任

- `index.py`：應用入口與首頁。
- `pages/1_🏠_房東入口.py`：房東營運面板、帳號／房源篩選、房源卡與風險資訊。
- `pages/2_🔍_租客入口.py`：租客端功能。
- `pages/3_📊_後台分析.py`：平台後台分析。
- `modules/ui_components.py`：全站共用 UI 樣式、設計 token 與元件。
- `modules/data_loader.py`：資料載入與快取。
- `modules/feature_engineering.py`：特徵處理、模型載入與預測。
- `modules/quadrant.py`：模型預估與真實檔期的象限／營運狀態判定。
- `modules/listing_detail.py`：共用房源詳情介面。
- `modules/geo_utils.py`：離線行政區與推估地址邏輯。
- `modules/calendar_analytics.py`、`modules/calendar_sections.py`：日曆與訂房檔期分析。
- `tests/`：目前主要涵蓋通知組合與平台分析邏輯。
