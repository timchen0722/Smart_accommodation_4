# 房源風險分析結果資訊架構改版設計

日期：2026-07-22

## 目標

在保留現有暖色系、白色圓角卡片、五欄桌面排版與行動版響應式行為的前提下，移除分類名稱與房源數量在統計卡片及摘要表格中的重複顯示，並讓統計卡片成為分類明細的主要操作入口。

本次同時統一兩個分類顯示名稱：

- `discount`：由「靠降價撐住」改為「預測風險與近期訂房不符」。
- `unknown`：由「檔期資料不足」改為「其他」。

內部穩定代碼 `alarm`、`hidden`、`discount`、`healthy`、`unknown` 保持不變，避免破壞既有資料、條件判斷、通知排序與報表生成；所有對使用者顯示的名稱、說明及建議均由中央映射取得。

## 資訊架構與互動

### 分類卡片

上方固定顯示五張分類卡片，依下列優先序排列：

1. 真警報
2. 隱形危機
3. 預測風險與近期訂房不符
4. 健康
5. 其他

每張卡片只顯示圖示、分類名稱與去重後房源數量。數量為零時仍保留卡片，使分類架構與操作位置穩定。

卡片以 Streamlit 原生按鈕實作並套用現有視覺樣式。點擊卡片後，以 `st.session_state` 記錄目前分類並重跑頁面；下方內容切換為該分類的分析與房源明細。提供低視覺權重的「顯示全部分析」控制，讓使用者返回預設狀態。

### 預設狀態

預設顯示全部分類的分析摘要，但不再顯示分類名稱與房源數量欄。每個唯一分析組合呈現：

- 分析判斷
- 主要原因
- 建議行動
- 查看對應房源控制

摘要資料依「分析判斷、主要原因、建議行動」去重；相同內容只顯示一次。

### 分類選取狀態

點擊分類卡片後，下方先顯示該分類的分析判斷、主要原因與建議行動，再顯示去重後的房源明細。明細欄位為：

- 房源名稱
- 行政區
- 模型風險
- 未來 90 天訂房率
- 主要原因
- 查看房源明細

「查看房源明細」使用 Streamlit 原生按鈕，展開或開啟該房源的精簡診斷內容。若房源沒有 90 天檔期資料，欄位顯示「—」，不推導為零。

## 分類文案

### 預測風險與近期訂房不符

說明：模型預測結果與近期實際訂房表現存在落差，可能受到近期促銷、價格調整、臨時訂單或市場變化影響。

建議行動：檢查近期訂單、價格變動、促銷活動及入住率，確認模型預測與實際營運狀況產生差異的原因。

主要原因：模型風險等級偏高，但近期檔期未呈現同等程度的空置訊號。

### 其他

說明：因資料日期範圍、欄位完整性或分析條件不足，目前暫時無法歸入主要風險分類。

建議行動：補充或確認資料後重新分析，暫不納入主要風險判斷。

主要原因：缺少可用的近期檔期資料，或資料未通過主要分類所需的完整性條件。

其他三類延用現有判斷語意，但將摘要拆分為「分析判斷、主要原因、建議行動」，避免同一段內容在不同欄位重複。

## 資料處理

### 唯一識別欄位

資料處理函式優先使用呼叫端指定的 ID 欄位；未指定時依序偵測 `listing_id`、`id`。找不到唯一識別欄位時明確拋出錯誤，不以資料列索引代替房源 ID。

### 去重與優先序

分類資料先計算 `quadrant_priority`，依優先序由高至低排序，再依唯一房源 ID 保留第一筆。優先序為：

`alarm < hidden < discount < healthy < unknown`

數字越小代表優先級越高。若同一房源因上游資料重複或多條件來源出現多列，最終只保留優先級最高的分類。

日曆資料在合併房源資料前先依 `listing_id` 去重，避免一對多合併放大房源列數。分類完成後再執行一次唯一房源去重，形成卡片、摘要、明細及匯出的共同資料來源。

### 數量守恆

卡片數量只從去重後的共同資料集計算，並驗證：

`所有分類卡片數量總和 == 去重後房源 ID 數量 == 所有分類明細唯一房源數量總和`

頁面不建立另一套獨立計數邏輯，以避免卡片與明細口徑漂移。

## 程式邊界

- `modules/quadrant.py`：中央分類映射、分析文案、優先序、日曆合併去重、房源分類去重、統計與一致性驗證。
- `pages/1_🏠_房東入口.py`：可點擊分類卡片、預設分析摘要、分類房源明細與互動狀態。
- `modules/report_builder.py`：沿用中央映射產生月報分類名稱、說明與建議。
- `modules/notify_center.py`：沿用中央映射顯示新名稱，保留既有通知條件與排序。
- `doc/06_v6新增功能報告.md`：更新仍被使用者閱讀的舊分類名稱與限制說明。
- 測試檔：驗證分類名稱、分類邊界、優先去重及數量守恆。

專案目前沒有獨立的房源風險分類 API。若未來加入 API，回傳的顯示名稱應直接取自同一中央映射，不另建文字常數。

## 錯誤與空值處理

- 缺少 `booked_rate_d90` 時歸為 `unknown`，顯示名稱為「其他」。
- 日曆分析產物不存在時，保留房源並歸入「其他」，不讓頁面失敗。
- 房源名稱缺失時以房源 ID 顯示。
- 90 天訂房率缺失時顯示「—」。
- 某分類沒有房源時卡片顯示 0，下方顯示無符合房源提示。
- 唯一識別欄位缺失時停止計數並顯示明確錯誤，避免產生錯誤統計。

## 驗證策略

### 單元驗證

- `discount` 與 `unknown` 的新名稱、說明及建議完全符合核准文案。
- 缺少檔期資料時分類為 `unknown`。
- 同一 ID 的多分類資料只保留最高優先級。
- 日曆重複 ID 不會使合併後房源數增加。
- 五類數量加總等於唯一 ID 數量。
- 各分類卡片數量等於該分類明細的唯一 ID 數量。

### 頁面驗證

- Streamlit AppTest 可載入房東入口。
- 預設狀態顯示分析摘要，不重複分類統計數量。
- 五張卡片皆可操作，切換後只顯示對應明細。
- 可返回全部分析。
- 桌面維持五欄，窄螢幕可水平捲動或自然堆疊，內容不溢出。

### 全域文字與回歸檢查

- 搜尋執行中程式、通知、圖表與匯出報表，確認不再顯示「靠降價撐住」或「檔期資料不足」。
- 確認通知觸發條件、模型分類規則、風險優先序及其他分頁功能未改變。
- 保留工作區原有未提交修改，只提交本功能涉及的檔案。
