## 五科計分規則（忠實 PDF）

### 交通 T = M + B（上限5）
- 捷運出口最近直線距離：≤400m→3、401–800→2、801–1200→1、>1200→0（上限3）
- 公車站牌最近直線距離：≤300m→2、301–600→1、>600→0（上限2）

### 生活 L = C + F + H + P（上限5）
- 超商/超市 500m 內 ≥1 → C=2
- 餐飲 800m 內 ≥5 個**去重**地點 → F=1（去重：name＋四捨五入座標）
- 診所 1km 內 ≥1 → H=1
- 公園 1km 內 ≥1 → P=1

### 價格（上限5）
- 比較組：同行政區 ＋ 同房型 ＋ 可住人數差 ≤1；中位數 Md。
- 樣本 <10：先放寬人數（同區＋同房型），標示「比較樣本較少」。
- 差異率 `D = (P − Md) / Md × 100%`：
  - D ≤ −10% → 5；−10%<D≤0% → 4；0%<D≤10% → 3；10%<D≤25% → 2；D>25% → 1
- 缺 Md 或比較組為空 → 標「資料不足」（不硬給分，排序時該科視為 0，但保留房源）。

### 口碑 R = min(A + N, C)（上限5）
- A（Airbnb 平均評分換算）：4.8–5.0→3、4.5–4.79→2.5、4.2–4.49→2、4.0–4.19→1、<4.0→0、無評分→資料不足
- N = `clamp(1 + P⁺ − P⁻, 0, 2)`；P⁺=正面則數/可分析則數、P⁻=負面則數/可分析則數；取**近20則**可分析評論（不足20用全部）
- C（評論數上限）：0則→不計分、1–4則→上限3、5–19則→上限4、≥20則→上限5
- 中立評論不加不減，但計入可分析總數。VADER/jieba 無法分析者排除、不當中立。

### 設備 E = m / k × 5（上限5）
- k＝使用者選的「希望設備」數（k≥1，預設帶入 Wi-Fi/冷氣/洗衣機）；m＝其中命中數。
- 設備字串先做同義詞比對（沿用/擴充頁面既有 `AMENITY_KW`）。

### 必要條件（第1關，不計分、硬篩選）
行政區、房型、預算上限、**必備設備**（缺少即排除）。

### 排序（優先分 Q）
1. Q＝最在意兩科分數相加（大→前）
2. 平手 → 五科總分 S
3. 再平手 → 評論數多者
4. 再平手 → 價格低者

### 總分區間
- 20–25 優先查看 / 15–19.9 值得考慮 / 10–14.9 普通 / 0–9.9 建議多比較

## 版面（照參考圖）

### 側欄
- 必要條件：行政區（複選）、房型（複選）、每晚預算（滑桿）
- 必備設備（複選，硬篩選）
- 最在意面向：多選限2科（交通/生活/價格/口碑/設備）→ 影響排序
- 希望設備（複選，計設備分；預設 Wi-Fi/冷氣/洗衣機）

### 主區
- 4 張 KPI 卡：符合條件房源數、中位價格、平均綜合分數(/25)、優先查看房源數（≥20分）
- 排序說明列（最在意兩科優先，再看總分）
- 房源卡清單（左）＋ 房源分佈地圖（右，下拉切換上色維度：綜合總分/交通/生活/價格/口碑/設備；圖例含區間色帶）
- 房源卡：縮圖＋名稱＋價格＋Airbnb評分＋綜合推薦 X/25＋五科小分（交通/生活/價格/口碑/設備）＋推薦理由（✓/△）＋三鈕（查看詳情／提出租房意願／加入收藏）
- 分頁（每頁 N 間）

### 詳情 Modal（`st.dialog`，4 分頁）
- 房源總覽：房源卡（地址/價格/評分/綜合推薦/推薦理由）＋五科雷達圖＋五科分卡
- 交通與生活圈：最近捷運出口(m)、最近公車站(m)、超商/餐飲/公園達標數＋小地圖
- 價格與設備：價格比較（本房源/同類中位/差異率）＋設備清單命中（✓/✗）
- 口碑與評論(NLP)：住客口碑分、情感分佈 donut、正面高頻關鍵詞 bar、有效分析 n/總 n、查看全部評論
