## 背景與共識

現況:`render_risk_management()`(`platform_sections.py:160`)用內層 `st.tabs()` 切三個子分頁
(📋 房源風險名單 / 👤 房東層級彙總 / ✉️ 派發輔導通知),掛在後台分析頁五大分頁的「🚨 風險管理」下。

需求:拿掉內層分頁切換,三段整合成同一頁「以聚焦房東為主軸」的連動戰情室。

已確認決策(grilling):
1. 走完整連動(路線 B),以「聚焦房東」為唯一跨區塊共享狀態。
2. 通知中心維持全平台批次派發本職;聚焦是**可選、預設全平台**。
3. 聚焦行為:名單/歸因/通知隨聚焦收斂;房東彙總排名表維持全平台當選擇器。
4. 控制項:房東彙總下拉(加「🌐 全平台」哨兵)當入口 + 頂部狀態列 +「回到全平台」鈕。
5. 篩選範圍:只共享聚焦房東,各區塊細部篩選維持獨立。
6. 版面:名單/彙總攤開;通知收進預設折疊 expander,標題動態顯示聚焦脈絡。

技術事實:`st.tabs` 本就每次全渲染 → 合併零效能代價;widget key 不衝突;
`render_notify_center(host_id=...)` 原生支援單一房東;`sec()` 非原生標題(無自動錨點,故不做跳轉列);
執行順序地雷用 `session_state.get()` 讀取(上段不放 widget)化解。
